# Creator Universe

> 창작자 매칭, 멀티 콘텐츠 유통, 코인 결제, 스마트 자동 정산을 하나로 연결하는 서브컬처 협업 플랫폼 MVP

[배포 사이트](https://project-limyoobins-projects.vercel.app) · [개인정보 처리방침](https://project-limyoobins-projects.vercel.app/privacy-policy.html) · [계정 삭제 안내](https://project-limyoobins-projects.vercel.app/account-deletion.html) · [백엔드 상태 확인](https://creator-universe-api-7qfc.onrender.com/health) · [GitHub Repository](https://github.com/limyoobin/Creator-Universe)

## 1. 프로젝트 개요

Creator Universe는 작가, 일러스트레이터, 성우, BGM 크리에이터가 함께 팀을 만들고 작품을 유통할 수 있는 **멀티 콘텐츠 협업 플랫폼**입니다.

서브컬처 창작 시장에서는 웹소설, 웹툰, 만화, 오디오드라마, 애니메이션처럼 여러 직군이 함께 만드는 콘텐츠가 빠르게 늘고 있습니다. 하지만 무명 창작자는 팀원을 찾기 어렵고, 수익 분배 기준이 불명확해 협업이 중단되는 경우가 많습니다.

Creator Universe는 이 문제를 해결하기 위해 **매칭, 결제, 지갑, 자동 정산, 팬 후원, 구독, 고객센터**를 하나의 흐름으로 연결했습니다.

## 2. 문제 정의

| 구분 | 문제 |
| --- | --- |
| 창작자 측면 | 외주 비용 부담, 팀원 섭외 어려움, 수익 분배 갈등 |
| 독자 측면 | 콘텐츠 형식이 흩어져 있고, 작품 탐색과 구매 경험이 분리됨 |
| 운영 측면 | 결제 내역, 정산 비율, 팀원 합의 기록을 투명하게 관리하기 어려움 |
| 접근성 측면 | 오디오 기반 콘텐츠와 배리어프리 UX가 부족함 |

## 3. 해결 방안

### 창작자 협업 매칭

창작자는 글, 그림, 목소리, BGM 등 자신의 직군과 포트폴리오를 등록할 수 있습니다. 다른 창작자는 프로필을 보고 채팅하거나 수익 지분율을 제안해 팀원을 모집할 수 있습니다.

### 스마트 자동 정산

프로젝트 팀원이 사전에 합의한 수익 분배율을 등록합니다. 독자가 코인으로 작품을 구매하면 플랫폼 수수료를 제외한 금액이 각 팀원의 지분율에 따라 자동 계산됩니다.

### 멀티 콘텐츠 유통

소설, 웹툰, 만화, 애니메이션, 오디오드라마, 믹스미디어 작품을 하나의 작품 페이지에서 탐색할 수 있습니다. 독자는 장르와 콘텐츠 형식을 조합해 원하는 작품을 찾을 수 있습니다.

### 팬덤 수익화

작품 구매 외에도 창작자 후원, 창작자 구독, 프리미엄 구독, 유료 포스트 흐름을 제공합니다. 창작자는 작품 밖에서도 팬덤 기반 수익을 만들 수 있습니다.

## 4. 핵심 기능

| 기능 | 설명 |
| --- | --- |
| 회원 시스템 | 회원가입, 로그인, 로그아웃, 아이디 찾기, 비밀번호 재설정, 계정 탈퇴 |
| 작품 탐색 | 장르 필터, 콘텐츠 형식 필터, 랭킹, 상세 페이지, 댓글/리뷰 |
| 매칭 | 창작자 프로필 등록/삭제, 포트폴리오 탐색, 채팅, 매칭 제안 |
| 지분 제안 | 수익 지분율을 포함한 매칭 제안과 수락 흐름 |
| 결제/지갑 | 코인 충전, 작품 구매, 지갑 잔액, 코인 이용 내역 |
| 자동 정산 | 플랫폼 수수료 차감 후 팀원 지분율에 따른 예상 정산 |
| 구독/후원 | 창작자 후원, 창작자 구독, 프리미엄 구독, 구독 해지 |
| 운영 지원 | 고객센터 문의, 사용자 신고, 챗봇, 알림센터 |

## 5. 서비스 차별화

| 비교 대상 | 기존 한계 | Creator Universe의 차별점 |
| --- | --- | --- |
| 외주 매칭 플랫폼 | 1회성 거래 중심, 협업 이후 수익 분배 관리가 어려움 | 팀 구성 이후 결제와 정산까지 연결 |
| 콘텐츠 유통 플랫폼 | 개인 창작물 유통 중심, 다자간 협업 정산이 약함 | 프로젝트 팀 단위 수익 분배 구조 제공 |
| 후원/구독 플랫폼 | 팬 후원은 가능하지만 작품 제작 협업 흐름은 약함 | 매칭, 작품, 팬덤 수익화를 한 서비스 안에서 처리 |

## 11. 로컬 실행

### 빠른 실행

Windows에서 바로 확인하려면 루트 폴더의 `start-creator-universe.cmd`를 실행합니다.

```text
Frontend: http://127.0.0.1:5173
Backend:  http://127.0.0.1:4000/health
```

이 방식은 PostgreSQL 없이 `scripts/demo-api.mjs` 데모 API를 사용합니다.

### 명령어로 실행

터미널 2개를 열고 아래 명령어를 각각 실행합니다.

```bash
npm run dev:demo-api
```

```bash
npm run dev:web
```

## 12. 실제 DB 연동 실행

1. 루트 의존성 설치

```bash
npm install
```

2. 프론트엔드 의존성 설치

```bash
npm --prefix frontend install
```

3. `.env.example`을 참고해 `.env` 생성

```env
DATABASE_URL="postgresql://USER:PASSWORD@HOST/DB?sslmode=require"
PORT=4000
```

4. Prisma 준비

```bash
npm run prisma:generate
npm run prisma:migrate
npm run prisma:seed
```

5. 백엔드 실행

```bash
npm run dev:api
```

6. 프론트엔드 실행

```bash
npm run dev:web
```

## 14. 주요 API

| Method | Endpoint | 설명 |
| --- | --- | --- |
| POST | `/api/auth/signup` | 회원가입 |
| POST | `/api/auth/login` | 로그인 |
| GET | `/api/auth/me` | 내 계정 확인 |
| POST | `/api/auth/logout` | 로그아웃 |
| POST | `/api/auth/check-username` | 아이디 중복 확인 |
| POST | `/api/auth/check-display-name` | 닉네임 중복 확인 |
| GET | `/api/creators` | 창작자 목록 |
| POST | `/api/creators/me/profile` | 내 매칭 프로필 등록 |
| DELETE | `/api/creators/me/profile` | 내 매칭 프로필 삭제 |
| GET | `/api/projects/:projectId` | 프로젝트 상세 |
| POST | `/api/settlements/content-purchase` | 콘텐츠 구매 및 자동 정산 |
| GET | `/api/projects/:projectId/settlement-dashboard` | 정산 대시보드 |
| GET | `/api/users/me/wallet` | 내 지갑 잔액 |
| POST | `/api/users/me/wallet/charge` | 코인 충전 |
| POST | `/api/chats/messages` | 채팅 메시지 전송 |
| POST | `/api/matching/requests` | 매칭 제안 생성 |

로그인이 필요한 API는 `Authorization: Bearer <token>` 헤더를 사용합니다.

## 15. 배포

- 프론트엔드는 Vercel에서 `main` 브랜치 기준으로 자동 배포됩니다.
- 백엔드는 Render에서 `render.yaml`을 기준으로 자동 배포됩니다.
- 데이터베이스는 Neon PostgreSQL을 사용합니다.
